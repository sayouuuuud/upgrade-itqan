## 6. مهمة 3: إصلاح السايدبار (الأهم — اقرأ بتركيز)

**الملف:** `components/dashboard-shell.tsx`

### 6.1 المبدأ

بدل ما الشيل يبني `rawConfig` بدمج الأقسام، يبني **config مختلف كلياً حسب `adminMode`**:

```ts
// بدل الدمج الحالي:
const rawConfig = isSuperAdminRole
  ? { ...baseConfig, sections: [...baseConfig.sections, ...getSuperAdminSections(t)] }
  : baseConfig

// المطلوب:
const rawConfig = isSuperAdminRole
  ? getConfigForMode(adminMode, t)   // 'super' | 'maqraa' | 'academy'
  : baseConfig                        // الأدوار الأخرى لا تتأثر إطلاقاً
```

### 6.2 سايدبار وضع المقرأة (maqraa)

هو **نفس** config الـ `admin` الحالي في `getRoleConfig` (السايدبار القديم بالظبط زي الصور: الرئيسية / المستخدمون والصلاحيات / العملية التعليمية / التفاعل / المجتمع / الإحصائيات والتقارير / الإعدادات / الأدوات المتقدمة / الحساب) **مع حذف البنود الآتية فقط** لأنها إعدادات موقع عامة تخص السوبر أدمن:

| البند يُحذف من وضع المقرأة | السبب |
|---|---|
| `/admin/homepage` (الصفحة الرئيسية) | تحكم في موقع الزوار، ليس المقرأة |
| `/admin/seo` (إعدادات SEO) | عام للموقع كله |
| `/admin/security` (الأمان) | عام للمنصة |
| `/admin/backup` (النسخ الاحتياطي) | عام للمنصة |

**كل الباقي يبقى**: المستخدمين، إدارة المشرفين، الدعوات، المقرئين، طلبات المقرئين، طلبات التدريس، أسئلة طلبات الانضمام، الطلاب، الإشراف، التلاوات، مسابقات الأكاديمية/المقرأة، لوحة المتصدرين، الشارات، مسارات الحفظ، مسارات التجويد، الجلسات والحجوزات، الحلقات، إعدادات البث والفيديو، طلبات الشهادات، مركز الشهادات، المحادثات، المنتدى وإدارته، صندوق الأسئلة الفقهية، المكتبة الفقهية، مكتبة الكتب وإدارتها، الإعلانات، التقارير، سجل الأنشطة، الإشعارات، قوالب البريد، إعدادات النظام (`/admin/settings` — إعدادات نظام المقرأة نفسها، مش الموقع)، التحكم بالوصول، البروفايل.

### 6.3 سايدبار وضع الأكاديمية (academy)

ابنِ أقسامه **بنسخ بنية الـ nav من `components/academy-dashboard-shell.tsx`** (دور `academy_admin` هناك) — نفس الأقسام والترتيب والأيقونات زي الصور المرفقة:
- الرئيسية: لوحة التحكم، التحليلات
- العملية التعليمية: الدورات، أرشيف الدورات، التصنيفات، المسارات التعليمية
- المستخدمون والصلاحيات: المدرسين، طلبات التدريس، أسئلة طلبات الانضمام، الطلاب، الإشراف، المستخدمين، الدعوات
- التفاعل: المسابقات، لوحة المتصدرين، الشارات
- الحلقات والبث: الحلقات، إعدادات البث والفيديو
- المجتمع: المنتدى والمقالات، إدارة المنتدى، صندوق الأسئلة الفقهية، المكتبة الفقهية، مكتبة الكتب، إدارة المكتبة، الإعلانات، مركز المحادثات
- الإعدادات: مركز الشهادات، التحكم بالوصول، إعدادات النظام (`/academy/admin/settings`)

**الروابط تفضل على `/academy/admin/...`** — الصفحات دي موجودة وشغالة، ممنوع تكرارها تحت `/admin`.

**يُحذف من وضع الأكاديمية:** أي بند إعدادات موقع عامة (لا يوجد حالياً في شيل الأكاديمية بنود homepage/seo/backup — فقط تأكد من عدم إضافتها).

### 6.4 سايدبار وضع السوبر أدمن (super)

**فقط** الأقسام الآتية (مبنية على `getSuperAdminSections` الحالية + الإحصائيات والمستخدمين):

- **الرئيسية:** لوحة التحكم الشاملة (`/admin`)، نظرة عامة على المنصة (`/admin/analytics`)
- **هوية الموقع:** الصفحة الرئيسية (`/admin/homepage`)، SEO (`/admin/seo`)، التصميم والألوان (`/admin/theme`)، الهوية البصرية (`/admin/branding`)، صفحات المحتوى الثابت (`/admin/content-pages`)
- **الحوكمة والصلاحيات:** المستخدمون (`/admin/users` — النظرة الموحدة عبر المنصتين)، إدارة الأدوار (`/admin/role-management`)، الأمان (`/admin/security`)، سجل التدقيق الموحد (`/admin/audit-log`)
- **إعدادات المنصة:** إعدادات الموقع العامة (`/admin/site-settings`)، التكاملات والخدمات (`/admin/integrations`)
- **الإشراف والمراقبة:** النسخ الاحتياطي (`/admin/backup`)
- **الحساب:** الإشعارات، البروفايل

**لا يظهر في وضع السوبر:** أي بند تشغيلي للمقرأة أو الأكاديمية (تلاوات، حلقات، دورات، مقرئين، مدرسين... إلخ). عايز يديرهم؟ يبدّل الوضع من السويتشر.

### 6.5 الـ Mode Switcher

- يبقى مكانه فوق السايدبار (زي الصور: "الأكاديمية / المقرأة" segmented control).
- يضاف له خيار ثالث "شاملة/سوبر" ليكون: **شاملة | المقرأة | الأكاديمية**.
- عند التبديل: يكتب الكوكي `ADMIN_MODE_COOKIE` (المنطق موجود في `admin-role-switcher.tsx` — راجعه وعدّل عليه بدل إعادة الاختراع) ثم يعمل refresh/navigation عشان السايدبار والداشبورد يتبدلوا.
- التبديل لوضع الأكاديمية ممكن يوجه لـ `/academy/admin` مباشرة (لأن صفحات الأكاديمية هناك) — **قرار للمنفذ**: إما توجيه مباشر لـ `/academy/admin` بشيلها القديم، أو عرض سايدبار الأكاديمية داخل `/admin` بروابط تشاور على `/academy/admin/...`. **الخيار الأول أبسط وأقل هشاشة ويستفيد من الشيل القديم السليم — نفّذه ما لم يكسر تجربة السوبر أدمن.**
- الـ switcher يظهر **فقط** للسوبر أدمن (`role === 'admin' || role === 'super_admin'`). أدمن المقرأة المحدود وأدمن الأكاديمية المحدود لا يشوفوا switcher.

### 6.6 ممنوعات صريحة في مهمة السايدبار

- ممنوع تعديل config أدوار `student` / `reader` / `student_supervisor` / `reciter_supervisor`.
- ممنوع تغيير ديزاين/ستايل الشيل نفسه (ألوان، عرض، حركة الطي) — التغيير في **محتوى البنود فقط** ومنطق اختيار الـ config.
- ممنوع لمس `components/academy-dashboard-shell.tsx` إلا للقراءة/النسخ.
- ممنوع لمس أي صفحة تحت `app/academy/admin/`.

### التحقق (إجباري بعد كل مهمة — استخدم agent-browser)

- سجّل دخول بسوبر أدمن وافتح `/admin`:
  - الوضع "شاملة": السايدبار يعرض أقسام الحوكمة فقط (قسم 6.4). لا يظهر أي بند مقرأة/أكاديمية.
  - بدّل لـ"المقرأة": السايدبار = سايدبار المقرأة القديم ناقص البنود الأربعة (homepage/seo/security/backup)، والداشبورد = الديزاين القديم.
  - بدّل لـ"الأكاديمية": سايدبار الأكاديمية القديم، والداشبورد = الديزاين القديم بتاع الأكاديمية.
- افتح بحساب أدمن مقرأة محدود (لو متاح): لا switcher، سايدبار المقرأة بدون بنود الموقع العامة.
- Screenshot لكل وضع وقارن بالصور المرفقة في المحادثة.
- تأكد من عدم وجود أخطاء console أو 404 في أي رابط سايدبار (اضغط كل بند).
