## 🌐 نظرة عامة على المنصة

منصة **إتقان** هي منصة إسلامية تعليمية شاملة تتكون من **كيانين مستقلين** يعملان تحت سقف واحد:

| الكيان | الوصف | الحالة |
|--------|--------|--------|
| 🟢 **المقرأة** (Quran Tahfeeth) | منصة تسميع وتقييم تلاوة القرآن الكريم | موجودة ويتم تطويرها |
| 🔵 **الأكاديمية** (Islamic Academy) | منصة تعليمية بنظام الفصول والدورات والدروس | تُبنى من الصفر |

> **المبدأ المعماري:** كل كيان مستقل بذاته تمامًا — كشقتين في نفس العمارة، كل واحدة قافلة بابها على نفسها. لكن المستخدم يمتلك مفتاح للشقتين إذا سُمح له.

## 🏗️ بنية الكيانين المستقلين

```
منصة إتقان (Itqan Platform)
├── 🟢 المقرأة (Quran/Tahfeeth) ─────── /student, /reader, /admin
│   ├── حسابات: طالب، مقرئ، مشرف طلاب، مشرف مقرئين، أدمن
│   ├── وظائف: تسميع، تقييم، حجز جلسات، شهادات
│   └── بنية: /app/student, /app/reader, /app/admin
│
├── 🔵 الأكاديمية (Academy/LMS) ────── /academy/*
│   ├── حسابات: طالب، أستاذ، مشرف محتوى، مسؤول فقه، ولي أمر، أدمن
│   ├── وظائف: دورات، دروس فيديو، مهام، حلقات، مسابقات، منتدى
│   └── بنية: /app/academy/student, /app/academy/teacher, /app/academy/admin
│
└── 🔗 الربط المشترك
    ├── نظام تسجيل موحد (Unified Auth)
    ├── نظام تبديل (Mode Switcher)
    └── أدمن بانل مزدوجة (Dual Admin)
```

## 🔐 سيناريو التسجيل واختيار المنصة

### السيناريو الأساسي: تسجيل طالب جديد

```
الطالب يفتح الموقع → يضغط "تسجيل جديد" (/register)
       ↓
يدخل: الاسم + الإيميل + كلمة المرور + الجنس
       ↓
🆕 [خطوة جديدة] اختيار المنصة:
   ○ المقرأة فقط (تسميع وتقييم القرآن)
   ○ الأكاديمية فقط (دورات ودروس)
   ○ الاثنين معًا ✅ (افتراضي)
       ↓
📧 إيميل كود OTP (صالح 24 ساعة)
       ↓
يدخل الكود في /verify
       ↓
✅ الحساب مفعّل → يتم توجيهه حسب اختياره:
   - المقرأة فقط → /student
   - الأكاديمية فقط → /academy/student
   - الاثنين → /student (مع زر Switch للأكاديمية)
```

### سيناريو تسجيل مقرئ

```
المقرئ يفتح /reader-register
       ↓
يدخل بياناته + مؤهلاته + يرفع ملف صوتي تجريبي
       ↓
📧 تفعيل الإيميل
       ↓
⏳ الطلب في حالة انتظار → الأدمن أو مشرف المقرئين يراجع
       ↓
✅ قبول → المقرئ يدخل /reader
❌ رفض → إشعار بالرفض مع السبب
```

### سيناريو تسجيل أستاذ أكاديمية

```
الأدمن يرسل دعوة للأستاذ عبر الإيميل (من لوحة الأكاديمية)
   أو
الأستاذ يتقدم بطلب انضمام من /academy/teacher-apply
       ↓
يملأ: البيانات + التخصص (قرآن/سيرة/فقه/عقيدة) + مؤهلات
       ↓
⏳ انتظار المراجعة من الأدمن
       ↓
✅ قبول → يدخل /academy/teacher
```

### سيناريو الدعوة من الأدمن

```
الأدمن يدخل لوحة الدعوات
       ↓
يختار: إيميل + الدور + المنصة (مقرأة/أكاديمية/الاثنين) + خطة تعليمية (اختياري)
       ↓
📧 إيميل دعوة يصل للمستهدف برابط شخصي
       ↓
المدعو يضغط → صفحة قبول → يُنشئ حساب أو يُضاف للخطة مباشرة
       ↓
⏰ الدعوة تنتهي بعد 7 أيام تلقائيًا
```

## 🔄 نظام التبديل (Switch) بين الكيانين

### كيف يعمل الـ Mode Switcher

الملف الحالي: `components/mode-switcher.tsx`

```
┌─────────────────────────────────────┐
│  🔄 تبديل الوضع                      │
│                                     │
│  🟢 المكتبة الصوتية (المقرأة)        │ ← الحالي
│     تلاوات وتسميع القرآن             │
│                                     │
│  🔵 الأكاديمية                       │ ← اضغط للتبديل
│     الدورات والدروس التفاعلية        │
│                                     │
└─────────────────────────────────────┘
```

| الدور | المقرأة | الأكاديمية | Switch متاح؟ |
|-------|---------|-----------|-------------|
| طالب مقرأة فقط | `/student` | ❌ | لا |
| طالب أكاديمية فقط | ❌ | `/academy/student` | لا |
| طالب مزدوج | `/student` | `/academy/student` | ✅ نعم |
| مقرئ فقط | `/reader` | ❌ | لا |
| مقرئ + أستاذ | `/reader` | `/academy/teacher` | ✅ نعم |
| أستاذ فقط | ❌ | `/academy/teacher` | لا |
| أدمن | `/admin` | `/academy/admin` | ✅ دائمًا |

> **ملاحظة:** الأدمن لديه دائمًا وصول للكيانين ويستطيع التبديل بينهما من أي مكان.

## 🛡️ التحكم في وصول المستخدمين

### صلاحيات الأدمن في التحكم بالوصول

```
الأدمن يدخل → لوحة إدارة المستخدمين
       ↓
يختار مستخدم → يحدد نوع الوصول:
   ○ المقرأة فقط
   ○ الأكاديمية فقط
   ○ الاثنين معًا
   ○ إيقاف الحساب كليًا
```

### جدول الوصول حسب الدور

| الدور | وصول المقرأة | وصول الأكاديمية | يتحكم فيه |
|-------|-------------|----------------|-----------|
| `student` | ✅ | ❌ | الأدمن |
| `academy_student` | ❌ | ✅ | الأدمن |
| `dual_student` | ✅ | ✅ | الأدمن |
| `reader` | ✅ | ❌ | الأدمن + مشرف المقرئين |
| `teacher` | ❌ | ✅ | الأدمن |
| `reader_teacher` | ✅ | ✅ | الأدمن |
| `parent` | ❌ | ✅ | الأدمن |
| `admin` | ✅ | ✅ | Super Admin |

### حقول قاعدة البيانات المطلوبة

```sql
-- إضافة حقول الوصول للمنصات في جدول المستخدمين
ALTER TABLE users ADD COLUMN has_quran_access BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN has_academy_access BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN platform_preference TEXT DEFAULT 'both'; -- 'quran' | 'academy' | 'both'
```

## 📝 ملاحظات تقنية مهمة

1. **Middleware:** كل مسار محمي — الطالب لا يدخل `/reader` والمقرئ لا يدخل `/admin`
2. **Session واحدة:** حساب واحد لكل مستخدم، الـ Switch لا يعمل logout/login
3. **فصل الجنسين:** يتم على مستوى العرض وليس على مستوى الحساب
4. **الـ Shell مستقل:** لكل كيان Shell خاص (قائمة جانبية + header)
5. **الباك إند جاهز لأغلب الميزات:** المطلوب الأساسي هو ربط الفرونت بالـ API الموجود
6. **الإشعارات لا تكسر التدفق:** إذا فشل الإشعار، العملية الأساسية تستمر
7. **أوقات الصلاة:** تظهر في Dashboard الطالب بالمقرأة + تذكيرات Cron
